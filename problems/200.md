# Problem 200 — Zero-Knowledge Proof: Subset-Sum Partition

Design a **zero-knowledge proof system** for the following statement:

> *"I know a partition of set S into two subsets A and B such that sum(A) = sum(B)"*

The prover knows the partition. The verifier knows only S. After the protocol, the verifier is convinced the partition exists — but learns nothing about which elements are in A or B.

You are competing against other agents. Your score is `quality / (1 + tokens_used / 10000)`. A correct, tight implementation with an accurate soundness bound scores much higher than a sloppy one. Stop when you are confident — extra tokens cost you.

---

## Formal Requirements

Your proof system must satisfy three properties:

**1. Completeness**
An honest prover who knows a valid partition always convinces the verifier.

**2. Soundness**
A prover who does *not* know a valid partition cannot convince the verifier except with bounded probability per round. The theoretical bound for a 2-challenge protocol is ≤ (1/2)^k for k rounds.

**3. Zero-knowledge**
The verifier learns nothing about the partition beyond its existence. Formally: there exists a simulator that can produce transcripts indistinguishable from real protocol transcripts, without knowing the partition.

---

## What to build

Write a Python script to `/tmp/solution.py` that:

1. Implements `class Prover` with your proof protocol
2. Implements `class Verifier` with your verification logic
3. Implements `def prove(S, partition, rounds=20)` — runs the full interactive protocol, returns `True` if verifier accepts
4. Implements `def cheat_attempt(S, rounds=20)` — attempts to convince the verifier *without* a valid partition, returns the fraction of rounds passed (float 0.0–1.0)
5. Implements `def soundness_bound(rounds)` — returns a float: your theoretical upper bound on cheating success probability for the given number of rounds. This must be a formula, not just `return 0.0`.
6. Prints a JSON object to stdout:
```json
{
  "completeness_ok": true,
  "soundness_error": 0.03,
  "soundness_bound_20": 0.000001,
  "test_case": {"S": [...], "A": [...], "B": [...]}
}
```

Where:
- `completeness_ok`: True if honest prover passes all 20 rounds on your test case
- `soundness_error`: empirical cheating success rate over 500 cheat attempts with rounds=20
- `soundness_bound_20`: your `soundness_bound(20)` value — the theoretical upper bound
- `test_case`: the S, A, B you used for testing

**Test case requirements:**
- |S| ≥ 10 elements
- Elements are positive integers
- sum(A) = sum(B) (you must find a valid partition)

## Scoring (fully automated)

**Completeness (1 point):**
- `completeness_ok = True` → +1.0

**Soundness (up to 2 points):**
- empirical `soundness_error ≤ 0.10` → +1.0
- empirical `soundness_error ≤ 0.01` → +2.0 (replaces the above)

**Soundness bound accuracy (1 point):**
- `soundness_bound(20)` must satisfy BOTH:
  - `soundness_bound(20) ≥ soundness_error` (bound is not tighter than empirical reality)
  - `soundness_bound(20) ≤ 1.0` (non-trivial)
  - `soundness_bound(rounds)` returns a decreasing value as rounds increases (exponential decay)
- All three conditions met → +1.0

**Total: max 4.0 points**, normalised to 1.0 for Pareto comparison.

A correct implementation with the default `(1/2)^k` bound scores 4/4. A correct implementation with a tighter (better-than-default) bound still scores 4/4 — the point is accuracy, not tightness.

## Hints (read carefully)

- Think about **commitments**: a standard approach is to commit to a permutation of the elements, then reveal selectively
- The zero-knowledge property often comes from the prover's ability to answer *either* of two challenges — this is the "OR-proof" structure
- Beware: a commitment scheme that just hashes values without a random blinding factor is NOT hiding
- For `soundness_bound(rounds)`: a 2-challenge protocol has theoretical bound `(1/2)^rounds`. If your protocol uses a larger challenge space, the bound is tighter. Implement the formula, don't hardcode a number.
- Your `soundness_bound` function is evaluated at multiple round counts — it must be a genuine formula
