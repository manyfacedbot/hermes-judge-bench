# Problem 200 — Zero-Knowledge Proof: Subset-Sum Partition

Design a **zero-knowledge proof system** for the following statement:

> *"I know a partition of set S into two subsets A and B such that sum(A) = sum(B)"*

The prover knows the partition. The verifier knows only S. After the protocol, the verifier is convinced the partition exists — but learns nothing about which elements are in A or B.

---

## Formal Requirements

Your proof system must satisfy three properties:

**1. Completeness**
An honest prover who knows a valid partition always convinces the verifier.

**2. Soundness**
A prover who does *not* know a valid partition cannot convince the verifier except with probability ≤ 1/2 per round. With k rounds, cheating succeeds with probability ≤ (1/2)^k.

**3. Zero-knowledge**
The verifier learns nothing about the partition beyond its existence. Formally: there exists a simulator that can produce transcripts indistinguishable from real protocol transcripts, without knowing the partition.

---

## What to build

Write a Python script to `/tmp/solution.py` that:

1. Implements `class Prover` with your proof protocol
2. Implements `class Verifier` with your verification logic
3. Implements `def prove(S, partition, rounds=20)` that runs the full interactive protocol and returns `True` if accepted
4. Implements `def cheat_attempt(S, rounds=20)` that attempts to convince the verifier *without* a valid partition, returning the fraction of rounds it passes
5. Prints a JSON object to stdout:
```json
{
  "completeness_ok": true,
  "soundness_error": 0.03,
  "zk_argument": "...",
  "test_case": {"S": [...], "A": [...], "B": [...]}
}
```

Where:
- `completeness_ok`: True if honest prover passes all rounds on your test case
- `soundness_error`: empirical cheating success rate over 1000 cheat attempts (should be ≤ 0.1 for k=20 rounds)
- `zk_argument`: a written argument (string, at least 100 words) explaining WHY your construction is zero-knowledge — what does the simulator do, why are transcripts indistinguishable?
- `test_case`: the S, A, B you used for testing

**Test case requirements:**
- |S| ≥ 10 elements
- Elements are positive integers
- sum(A) = sum(B) (you must find a valid partition)

---

## Scoring

This problem is scored on three dimensions by the judge:

**Mechanical (automated):**
- `completeness_ok = True` → 1.0 points
- `soundness_error ≤ 0.10` → 1.0 points  
- `soundness_error ≤ 0.01` → 2.0 points (bonus for tight soundness)

**Qualitative (judge-evaluated, 0–3 points):**
The judge evaluates your `zk_argument` on:
- Does the simulator description actually work? (Can it produce valid transcripts without the witness?)
- Is the zero-knowledge claim vacuous, partial, or genuine?
- Are there hidden leaks (e.g. does your commitment scheme leak information about which elements were committed?)

**Final score:** `total_points / (1 + tokens_used / 10000)`

A trivially correct but shallow ZKP scores ~2/5. A genuinely sound construction with a valid ZK argument scores ~5/5. The judge's ability to distinguish these is what this problem tests.

---

## Hints (read carefully)

- Think about **commitments**: a standard approach is to commit to a permutation of the elements, then reveal selectively
- The zero-knowledge property often comes from the prover's ability to answer *either* of two challenges — this is the "OR-proof" structure
- Beware: a commitment scheme that just hashes values without a random blinding factor is NOT hiding
- Your `zk_argument` will be scrutinised — vague claims like "it's ZK because the verifier only sees commitments" will score poorly
